# 前后端功能对齐文档

## 概述

本文档记录了前端（user-client）和后端（agent-framework）功能对齐的工作，包括新增的用户端后端功能。

## 完成的工作

### 1. 用户认证系统（新增）

创建了完整的用户认证系统，包括：

**文件：** `agent_framework/gateway/auth.py`

**功能：**
- 用户注册（`/api/auth/register`）
- 用户登录（`/api/auth/login`）
- 获取当前用户信息（`/api/auth/me`）
- 用户登出（`/api/auth/logout`）
- 刷新访问令牌（`/api/auth/refresh`）

**技术实现：**
- JWT认证（使用PyJWT）
- 密码加密（PBKDF2-HMAC-SHA256）
- 用户管理器（UserManager）
- 访问令牌和刷新令牌机制

### 2. 会话管理API（新增）

创建了会话管理API端点，与前端agentApi对齐。

**文件：** `agent_framework/gateway/user_api.py`

**功能：**
- 获取用户会话列表（`/api/sessions`）
- 创建新会话（`/api/sessions`）
- 获取会话详情（`/api/sessions/{session_id}`）
- 删除会话（`/api/sessions/{session_id}`）

### 3. API集成

将用户端API集成到主API应用中。

**修改文件：** `agent_framework/gateway/api.py`

**集成内容：**
- 导入用户端API模块
- 创建SessionManager实例
- 调用`create_user_api()`注册所有用户端路由
- 添加CORS中间件支持

### 4. 依赖更新

**修改文件：** `agent_framework/pyproject.toml`

**新增依赖：**
- PyJWT>=2.8.0（JWT认证）
- email-validator（邮箱验证）

### 5. 前端环境配置

**新增文件：** `user-client/.env.local`

**配置内容：**
```
NEXT_PUBLIC_API_URL=http://localhost:18790
NEXT_PUBLIC_GATEWAY_URL=http://localhost:18790
NEXT_PUBLIC_GATEWAY_WS_URL=ws://localhost:18790/ws
```

## 架构说明

### 用户端后端架构

```
agent_framework/gateway/
├── auth.py           # 用户认证模块
│   ├── User          # 用户模型
│   ├── UserManager   # 用户管理器
│   ├── JWT相关函数   # 令牌创建和验证
│   └── 密码加密函数  # PBKDF2-HMAC-SHA256
│
├── user_api.py       # 用户端API路由
│   ├── 认证路由      # /api/auth/*
│   └── 会话路由      # /api/sessions/*
│
├── session.py        # 会话管理器（已存在）
│
└── api.py            # 主API应用
    └── 集成用户端API
```

### 认证流程

1. **注册流程：**
   - 用户提交用户名、邮箱、密码
   - 系统验证唯一性
   - 密码加密存储
   - 返回访问令牌和用户信息

2. **登录流程：**
   - 用户提交用户名和密码
   - 系统验证凭据
   - 创建访问令牌和刷新令牌
   - 返回令牌和用户信息

3. **认证保护：**
   - 使用Bearer Token认证
   - JWT令牌验证
   - 用户权限检查
